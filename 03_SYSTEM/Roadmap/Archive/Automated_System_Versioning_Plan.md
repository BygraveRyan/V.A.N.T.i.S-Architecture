## 1. OBJECTIVE
Implement a passive, zero-friction versioning engine that automatically increments the `Version: X.Y.Z` metadata in core system documents whenever they are modified. This ensures architectural changes are tracked, auditable, and synced with the Machine Layer (Logs).

---

## 2. ARCHITECTURAL COMPONENTS

### A. The Registry (`03_SYSTEM/Protocols/CORE_SYSTEM_REGISTRY.md`)
A central "Watch List" that categorizes system files. This prevents the versioning logic from being hardcoded and allows the user to easily add/remove files from the tracking loop.

| Tier | Scope | Examples | Versioning Logic |
| :--- | :--- | :--- | :--- |
| **Tier 1: Engine** | Root Protocols | `GEMINI.md`, `RULES.md` | `Major.Minor` (e.g., 1.1 -> 1.2) |
| **Tier 2: Protocols** | `03_SYSTEM/Protocols` Specs | `METADATA_SCHEMA.md` | `Major.Minor` (e.g., 1.0 -> 1.1) |
| **Tier 3: UI/Logic** | Templates/Dashboards | `GALAXY_NOTE_TEMPLATE.md` | `Patch` (e.g., 1.0.0 -> 1.0.1) |

### B. The Incrementer Hook (`.gemini/hooks/version-incrementer.js`)
A Node.js script triggered during the **VALIDATE -> LOG** cycle.
1. **Detect**: Scans the list of files modified in the current turn.
2. **Match**: Checks if any modified files are listed in the `CORE_SYSTEM_REGISTRY.md`.
3. **Update**: 
   - Locates `Version: X.Y.Z`.
   - Increments based on Tier logic.
   - Refreshes `Updated: YYYY-MM-DD` if present.

### C. The Skill Integration (`.gemini/skills/audit-logger/`)
The `audit-logger` will be updated to execute the incrementer hook *before* finalizing the audit log. This guarantees that the version jump is recorded in the log itself.

### Phase 1: Standardization (Manual)
1. **Audit Gaps**: Ensure `CHANGELOG.md` and `README.md` (excluding README per user request) have a `Version: 1.1` header.
2. **Sanitize Headers**: Ensure all files in `03_SYSTEM/Protocols/` use the standard `Version: X.Y` or `Version: X.Y.Z` format.

### Phase 2: Configuration
1. **Create Registry**: Author `03_SYSTEM/Protocols/CORE_SYSTEM_REGISTRY.md`.
2. **Define Rules**: Establish the RegEx patterns for version detection.

### Phase 3: Automation
1. **Scripting**: Develop `.gemini/hooks/version-incrementer.js`.
2. **Wiring**: Update the `GEMINI.md` Logging Protocol to include the mandatory versioning check.

## 4. VERIFICATION & SAFETY
- **Dry Run Mode**: The script will support a `--dry-run` flag to log proposed changes without writing to disk.
- **Rollback**: Since V.A.N.T.i.S. is Git-backed, any erroneous increments can be reverted via `git checkout`.
- **Exclusion**: Files in `Archive/` folders are explicitly ignored to prevent legacy versioning.
