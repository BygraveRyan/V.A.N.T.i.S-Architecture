## 🏗️ THE TWO-LAYER ARCHITECTURE

### LAYER A: THE PRIVATE "PULSE" (BACKGROUND BACKUP)
*   **Goal:** Zero-friction, 100% vault synchronization for cross-device use.
*   **Mechanism:** A background Git sync that commits and pushes directly to the `main` branch of the Private Core.
*   **Protocol:** 
    *   *Trigger:* On session close (via `/eod` or a post-session hook) or manually via `/gh:pulse`.
    *   *Commit:* `chore(sync): [timestamp] vault pulse sync`.
    *   *Destination:* Private `BygraveRyan/V.A.N.T.i.S` (main branch).

### LAYER B: THE "MILESTONE BRIDGE" (PORTFOLIO STORYTELLING)
*   **Goal:** A high-signal, "polished" history of architectural upgrades.
*   **Mechanism:** A PR-driven workflow that mirrors ONLY whitelisted "Engine" files.
*   **Protocol:**
    *   *Trigger:* When V.A.N.T.i.S. is performing an "Upgrade" task (e.g., Phase 4 of Tripartite).
    *   *Workflow:*
        1.  `git checkout -b feat/upgrade-name` (Private branch).
        2.  Execute changes + log generation.
        3.  `git commit` with Why/How/Impact.
        4.  `/gh:publish` creates a matching branch and PR on the **Public Mirror**.
    *   *Destination:* Public `BygraveRyan/V.A.N.T.i.S-Architecture`.
