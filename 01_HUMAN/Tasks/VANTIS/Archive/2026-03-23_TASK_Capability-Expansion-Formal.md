## 1. Background & Motivation
V.A.N.T.i.S. operates with a powerful set of specialized skills (e.g., `precedent-detection`, `audit-logger`, `concept-extraction`). Currently, agents often wait for explicit user instruction to load these skills, or rely on localized, inconsistent triggers within individual agent profiles. Furthermore, while Rule 11 mandates "Branched Evolution" and planning for systemic upgrades, there is no standardized protocol or skill to enforce *how* an architectural plan is drafted, leading to ad-hoc, inconsistent planning (like the static note previously generated).

This expansion aims to:
1. Shift from **Reactive** to **Proactive** skill activation via a centralized "Skill Trigger Matrix" in the Intelligence Layer.
2. Formalize system planning by creating the **Architectural Designer** skill, ensuring all complex changes follow a rigorous, standardized design template.
3. Automate **Knowledge Lineage**, defining clear promotion paths from AI synthesis to permanent Galaxy concepts.

## 2. Scope & Impact
- **Modified Core Files**: `GEMINI.md` (adding Skill Trigger Matrix), `AGENTS.md` (updating global mandates to consult the matrix).
- **New Capabilities**: Creation of `.gemini/skills/architectural-designer/SKILL.md`.
- **Replaced Files**: This task note replaces the previous static proposal `01_HUMAN/Tasks/2026-03-23_TASK_Capability-Expansion-Proposal.md`.
- **Impact**: Increased autonomy, lower turn-counts (as agents load skills instinctively), and higher-fidelity structural upgrades.

## 3. Proposed Solution

### A. The Skill Trigger Matrix (GEMINI.md Update)
We will introduce a "Skill Trigger Matrix" section in `GEMINI.md` under "Operational Behaviour" or "Skill Loading". This acts as a centralized logic gate.
**Matrix Concept**:
- **Intent**: Modify System/Architecture -> **Trigger**: `architectural-designer`
- **Intent**: Write LinkedIn Post -> **Trigger**: `linkedin-content-generation`, `precedent-detection`
- **Intent**: Process 01_HUMAN/Inbox -> **Trigger**: `inbox-processor`, `concept-extraction`
- **Intent**: Execute Git Branching, Commits, or PRs -> **Trigger**: `github-ops`
- **Intent**: Finalize a turn involving file modifications -> **Trigger**: `audit-logger`

### B. The Architectural Designer Skill (including Post-Plan Sync)
We will create a new specialized skill that takes over whenever a user requests a system change, refactor, or complex multi-file task.
**Skill Mandates (`.gemini/skills/architectural-designer/SKILL.md`)**:
- MUST enforce the creation of an "Architectural Upgrade Plan" using a specific markdown template in `01_HUMAN/Tasks/` or Plan Mode.
- **NEW: Post-Plan Sync Hook**: The skill MUST mandate that any plan generated in `plan_mode` is mirrored/written to `01_HUMAN/Tasks/` upon approval to ensure permanent auditability.
- MUST mandate the mapping of dependencies (which core files are affected).
- MUST enforce Rule 11 (Branched Evolution) as the final step of the planning phase.
- Replaces ad-hoc list-making with structured engineering design.

### C. Knowledge Lineage Automation
We will define the formal path in `METADATA_SCHEMA.md` or a lightweight task hook to transition a note from `status: ai-candidate` (in `02_MACHINE`) to `status: ai-assisted` (in `01_HUMAN/Knowledge/Galaxy`) upon human approval, ensuring the "Galaxy Protection" rule is maintained but not bottlenecked.
