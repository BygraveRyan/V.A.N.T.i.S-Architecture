# Verification Skill

Version: 1.0.0
Description: Multi-phase verification pipeline (Lint -> Protocol -> Security -> PR Report).
Trigger: Manual (/verify) or automated via PR hooks.

---

## 🏛️ OVERVIEW
The Verification Skill is the final quality gate for V.A.N.T.i.S. changes. It ensures that all modifications comply with established standards, security rules, and architectural protocols.

---

## 🚀 VERIFICATION PHASES

### 1. Lint & Syntax Check
- **Purpose**: Ensure code quality and consistency.
- **Action**: Run available project linters (e.g., `shellcheck`, `ruff`).
- **Failure**: Block merge.

### 2. Protocol Compliance (v1.5)
- **Purpose**: Ensure metadata and system structure are sound.
- **Checks**:
    - Metadata v1.5 present in new knowledge nodes.
    - ASV Reflex version increments on protocol files.
    - Directory routing complies with `VAULT_MAP.md`.
- **Failure**: Block merge.

### 3. Security & Safety Scan
- **Purpose**: Detect secrets, dangerous commands, or unauthorized access.
- **Checks**:
    - No API keys or tokens in cleartext.
    - No destructive shell commands (`rm -rf /`, etc.).
    - Guardian review required for network operations.
- **Failure**: Block merge.

### 4. PR Report Generation
- **Purpose**: Summarize findings for human review.
- **Action**: Compile results into a structured "Verification Report."

---
