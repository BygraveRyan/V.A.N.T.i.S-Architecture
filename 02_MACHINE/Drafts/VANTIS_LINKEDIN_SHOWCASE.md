## 🏗️ THE "WHY": Beyond the Chat Window
Traditional AI interactions suffer from two terminal flaws: **Context Fragmentation** and **Chat Limit Friction**. When a session ends or the context window hits its ceiling, the "intelligence" resets. The agent forgets the architecture, the pending tasks, and the nuance of previous decisions.

**V.A.N.T.i.S. (Vectorised Astral Navigation & Thought Intelligence System)** solves this by treating the **Local Git Repository** as the "Save Game" for human-AI collaboration. It is not an assistant; it is a **6-Layer Cognitive Infrastructure** that turns a markdown vault into a persistent, compounding intelligence center.

---

## ⚙️ THE "HOW": Deterministic State & Internal Plumbing
The secret to V.A.N.T.i.S. isn't the model—it's the **Internal Plumbing**. The system uses a sophisticated hook-and-command architecture to ensure 100% state preservation.

### 1. Deterministic Resumption (The "Save Game")
*   **`.resume` / `repo-map.sh`**: Every session begins with a mandatory orientation hook. The system scans the repository, reads the latest `session-state_YYYY-MM-DD.md`, and rebuilds its working memory of active objectives, next steps, and blockers.
*   **`.eod` / `compile-audit-log.sh`**: At session end, the system performs a "Master Audit." It serializes all modifications, updates the task dashboard, and generates a deterministic state file for the next agent (or human) to pick up.

### 2. The 6-Layer Architecture
1.  **Identity Layer (`BOOT_IMAGE.md`)**: Core persona and fundamental mission.
2.  **Intelligence Layer (`GEMINI.md`)**: Central nervous system; governs routing and tool prioritization.
3.  **Agent Layer (`AGENTS.md`)**: Specialized roles (Architect, Curator, Researcher) with scoped permissions.
4.  **Skill Layer (`.gemini/skills/`)**: Modular, lazy-loaded technical capabilities (GitHub Ops, Concept Extraction).
5.  **Memory Layer (`vault/`)**: Human-curated knowledge graph (The Galaxy) following strict `METADATA_SCHEMA v1.5`.
6.  **Logging Layer (`logs/`)**: Immutable, high-fidelity audit trail of every turn and decision.

### 3. Automated System Versioning (ASV)
*   **`version-incrementer.js`**: A native hook that monitors Tier 1–5 files (Protocols, Skills, Core Config). Upon modification, it automatically increments system versions (Major.Minor.Patch) and updates the `CORE_SYSTEM_REGISTRY`, ensuring the architecture evolves with the rigor of a production codebase.

## 🛠️ OPERATIONAL CAPABILITIES: Agentic Execution
V.A.N.T.i.S. moves beyond "chatting" into native **Agentic Execution** through a custom slash-command suite:

*   **`gh/pr` & `gh/issue`**: Native GitHub governance. The system reviews its own PRs, manages its issue board, and monitors CI/CD runners via the `gh` CLI.
*   **`gh/publish` & `gh/pulse`**: Automated data dissemination. "Pulse" handles silent background backups, while "Publish" sanitizes the engine for public portfolio mirrors.
*   **`inbox`**: Automated information routing. Raw notes are clarified and moved to Projects, Tasks, or the Galaxy based on the `INBOX_PROCESSING_PROTOCOL`.

## 🛡️ RULES OF ENGAGEMENT: 100% Auditability
To prevent "hallucination drift" and ensure system safety, V.A.N.T.i.S. operates under strict protocols:

*   **`AGENT_PERMISSIONS`**: Least-privilege access. A "Strategist" can read the Galaxy but cannot write to system protocols.
*   **`SIGNAL_PROTOCOL`**: Inter-agent coordination. Multi-agent handoffs are managed via explicit signals (`WAIT`, `SYNC`, `DONE`), preventing collisions in a shared vault.
*   **`Log Reflex`**: The "Gold Standard" of auditing. No turn may conclude without generating a high-fidelity log containing the reasoning summary, files modified, and outcome.
